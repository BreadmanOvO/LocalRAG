# Center-based 3D Object Detection and Tracking (CenterPoint)

## 概述

CenterPoint是由UT Austin和Motional联合提出的基于中心点的3D目标检测与跟踪方法，发表于CVPR 2021。该方法将2D检测领域CenterNet的核心思想扩展到3D空间，将3D目标检测建模为中心点检测问题，通过预测目标中心点的热力图（heatmap）以及回归中心点处的3D属性来完成检测任务。CenterPoint进一步利用中心点的速度估计实现帧间目标关联，构建了完整的3D检测与跟踪联合框架。该方法在Waymo和nuScenes两大自动驾驶基准上均取得了最优性能，成为LiDAR-based 3D感知的经典之作。

## 核心贡献

1. **中心点表示的3D检测**：将3D bounding box表示为其中心点在BEV平面上的位置以及该点处回归的3D属性（尺寸、高度、朝向角、速度等）。这种表示方式避免了传统anchor-based方法中复杂的anchor设计和方向分类问题。

2. **两阶段精炼框架**：第一阶段通过CenterNet风格的编码器-解码器网络预测粗略的中心点和属性；第二阶段从第一阶段的预测中心点处提取精确的体素特征，对3D框参数进行精炼，显著提升了定位精度。

3. **检测-跟踪联合优化**：利用CenterPoint第一阶段预测的速度向量将目标中心点传播到下一帧，通过最近邻匹配实现简单而高效的目标跟踪，无需复杂的关联算法（如匈牙利匹配或卡尔曼滤波）。

4. **通用性强**：CenterPoint的架构设计具有良好的通用性，可无缝支持多种传感器输入（纯LiDAR、多LiDAR融合）和多种Backbone网络（VoxelNet、PointPillars等）。

## 方法架构

### Voxel特征编码

CenterPoint首先将输入的LiDAR点云进行体素化（Voxelization）。点云空间被均匀划分为小的体素单元，每个体素内的点通过Voxel Feature Encoding（VFE）层进行特征聚合。CenterPoint默认采用动态体素化策略，仅对非空体素进行计算，大幅降低了计算开销。

编码后的体素特征被重新组织为BEV特征图。具体实现中，CenterPoint支持两种体素编码方式：一是基于3D稀疏卷积的VoxelNet（精度更高），二是基于PointPillars的柱状编码（速度更快）。3D稀疏卷积方案通过若干稀疏卷积层逐步降低Z轴分辨率、增加通道数，最终在Z轴上进行压缩，生成2D BEV特征图。

### 2D Backbone与特征金字塔

BEV特征图通过标准的2D卷积网络进行特征提取。CenterPoint采用了带有FPN的多尺度架构，包含多个下采样阶段（通过stride=2卷积）和上采样路径（通过反卷积），最终将多尺度特征融合为统一的高分辨率特征图。

### 第一阶段：中心点检测

**热力图预测**：对于每个类别，预测一张BEV热力图，其中峰值位置对应目标的中心点。热力图通过一个3x3卷积层后接sigmoid激活函数生成，使用改进的Focal Loss进行监督。高斯核被用来生成GT热力图，将每个目标的中心点扩展为一个高斯分布，其方差与目标尺寸成正比。

**属性回归**：在特征图的每个位置，通过并行的回归头预测以下属性：
- **中心偏移**（offset）：目标中心点相对于网格坐标的亚像素偏移。
- **Z轴高度**（z）：目标中心点的Z坐标。
- **3D尺寸**（size）：长、宽、高。
- **朝向角**（rotation）：使用sin/cos编码的朝向角。
- **速度**（velocity）：目标在X和Y方向上的速度分量，通过前后帧GT中心点的位移计算得到。

### 第二阶段：特征精炼

第一阶段产生一组粗略的3D框预测。对于每个预测框，CenterPoint从BEV特征图上提取其中心点处的特征，并聚合该点周围一个局部区域的特征（通过RoI-like操作）。聚合后的特征通过一个小型网络对3D框的7个自由度参数（x, y, z, l, w, h, θ）进行精炼回归。

第二阶段的关键设计是特征提取方式：不同于图像检测中的RoI Pooling，CenterPoint直接在BEV特征图上以预测中心点为锚点，采样其周围K个最近邻特征点，通过注意力加权聚合为精炼特征。这种设计保持了计算的高效性。

### 跟踪模块

CenterPoint的跟踪模块极为简洁：

1. **速度传播**：利用第一阶段预测的速度向量，将当前帧的检测中心点传播到下一帧的预测位置。
2. **最近邻关联**：对于传播后的预测位置与下一帧的检测结果，计算两两之间的L2距离，通过贪心最近邻匹配建立帧间目标关联。
3. **生命周期管理**：对每个track维护一个计数器，连续若干帧未被匹配的track被标记为丢失并移除。

这种基于中心点速度的跟踪方法避免了传统多目标跟踪中复杂的外观模型和运动模型设计，在保持高跟踪精度的同时实现了极低的计算开销。

### 损失函数

总损失函数为：

L = L_heatmap + L_attr + L_refine

其中L_heatmap为改进的Focal Loss（对正样本使用高斯分布软标签）；L_attr为第一阶段属性回归的Smooth L1 Loss（对前景点施加）；L_refine为第二阶段精炼回归的Smooth L1 Loss。

## 实验结果

在**Waymo Open Dataset**上，CenterPoint取得了当时最优的性能。在Vehicle检测任务上，CenterPoint达到76.5% mAPH（L2难度），显著超越了之前的最优方法如StarNet（69.2% mAPH）。在Pedestrian和Cyclist类别上，CenterPoint同样取得了领先性能。

在**nuScenes**数据集上，CenterPoint（基于VoxelNet Backbone）取得了67.3% NDS和60.3% mAP，大幅超越了PointPillars（45.3% NDS）和SECOND等方法。加入第二阶段精炼后，NDS进一步提升至68.4%。

在**跟踪任务**上，CenterPoint在nuScenes上取得了62.3% AMOTA（Average Multi-Object Tracking Accuracy），在无需任何外观特征的情况下超越了AB3DMOT等传统跟踪方法。

消融实验表明：第二阶段精炼贡献了约2-3%的mAP提升；速度预测对跟踪性能至关重要（去除后AMOTA下降约10%）；使用更大的Backbone和更长的训练周期可进一步提升性能。

## 局限性与讨论

1. **计算量较大**：CenterPoint（基于VoxelNet）的计算量高于PointPillars等轻量方法，在边缘设备上的实时部署面临挑战。

2. **对稀疏目标的检测**：在点云极度稀疏的远距离区域，中心点热力图可能无法产生足够强的响应，导致漏检。

3. **速度估计的局限**：速度预测依赖于前后帧的GT标注，在高速运动或遮挡频繁的场景下，速度估计可能存在较大误差，影响跟踪质量。

4. **纯LiDAR局限**：与PointPillars类似，CenterPoint仅使用LiDAR输入，无法感知颜色和纹理信息。后续工作TransFusion等方法在此方向上进行了LiDAR-Camera融合的探索。

CenterPoint以其简洁优雅的中心点表示、高效的两阶段精炼和简洁的跟踪方案，成为3D检测与跟踪领域的里程碑工作，至今仍是许多自动驾驶感知系统的核心组件。
