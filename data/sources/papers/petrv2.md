# PETRv2: A Unified Framework for 3D Perception from Multi-camera Images

## 概述

PETRv2是由上海交通大学和商汤科技联合提出的多相机3D感知统一框架，发表于ECCV 2022。作为PETR（Position Embedding Transformation）的升级版本，PETRv2在保留PETR核心的位置编码变换思想基础上，引入了时序对齐机制、特征引导的位置编码以及多任务学习能力，构建了一个同时支持3D目标检测、BEV分割和3D车道线检测的统一感知框架。PETRv2代表了纯视觉BEV感知从单任务向多任务、从单帧向时序建模演进的重要里程碑。

## 核心贡献

1. **时序对齐的位置编码（Temporal-Aligned Position Encoding）**：提出了一种将历史帧的3D位置编码与当前帧对齐的机制。通过对历史帧的reference points应用ego-motion变换，将历史帧的3D坐标系统一到当前帧坐标系下，使得模型能够有效利用时序信息提升3D检测精度。

2. **特征引导的位置编码（Feature-Guided Position Encoding, FGPE）**：引入图像特征来增强3D位置编码的生成。传统方法仅使用相机参数生成固定的3D位置坐标，而FGPE通过可学习的特征变换，使位置编码能够自适应地感知图像内容，提升了位置编码的表达能力和准确性。

3. **多任务统一架构**：PETRv2在统一的Transformer架构下同时支持3D目标检测、BEV语义分割和3D车道线检测三项任务，通过共享的编码器和任务特定的解码头实现多任务学习，展现了Transformer架构在多任务3D感知中的灵活性。

4. **改进的收敛策略**：通过引入辅助深度监督和优化训练策略，显著加速了模型的收敛过程，解决了原始PETR训练收敛慢的问题。

## 方法架构

### 位置编码生成

PETRv2继承了PETR的核心思想，将3D空间中的位置信息编码到Transformer的attention机制中。具体而言，对于每个相机视角，首先在相机坐标系下生成一组3D采样点（沿射线均匀分布），这些3D点通过相机外参变换到世界坐标系，再通过一个MLP网络映射为位置编码向量。

**特征引导的位置编码（FGPE）**在此基础上进行了关键改进。原始PETR的位置编码仅依赖几何变换，是静态的。FGPE额外引入了图像特征图的全局信息，通过一个轻量级的卷积网络提取特征描述子，与几何位置编码进行逐元素相加，生成包含语义信息的动态位置编码。这使得模型能够根据场景内容自适应调整位置编码，例如在远距离区域增强深度方向的编码精度。

### 时序融合模块

时序融合是PETRv2相比原始PETR最重要的改进之一。该模块维护一个历史帧缓存队列（通常存储最近1-2帧的信息）。对于每一帧历史信息，模型记录其对应的3D位置编码和编码器输出特征。

在融合时，首先利用自车运动信息（ego-motion）将历史帧的3D reference points变换到当前帧坐标系下，实现几何层面的对齐。随后，对齐后的历史位置编码与当前位置编码拼接，共同作为Transformer decoder的key和value参与cross-attention计算。这种设计使模型能够在3D空间中准确地关联当前帧与历史帧的信息，有效利用时序线索进行运动估计和目标跟踪。

### 多任务解码头

PETRv2的Transformer decoder输出统一的BEV-like特征表示，该表示被送入不同的任务特定解码头：

- **3D检测头**：沿用PETR的集合预测范式，通过FFN预测3D bounding box的7个自由度参数（中心、尺寸、朝向角）和类别概率。
- **BEV分割头**：将decoder输出的特征通过一个轻量级卷积网络映射到BEV平面，进行语义分割（如道路、人行道、车道线等）。
- **3D车道线检测头**：通过专门设计的车道线解码器，预测3D空间中的车道线点序列。

### 损失函数

总损失函数为各任务损失的加权和：

L = λ_det * L_det + λ_seg * L_seg + λ_lane * L_lane

其中L_det包含Focal Loss（分类）和L1 Loss（3D框回归）；L_seg使用Cross-Entropy Loss进行BEV语义分割；L_lane采用车道线点回归损失。此外，训练过程中引入深度估计的辅助损失L_depth，通过监督隐式深度分布加速模型收敛。

## 实验结果

在nuScenes 3D检测任务上，PETRv2取得了显著的性能提升。在nuScenes val set上，PETRv2达到38.1% NDS和40.4% mAP，相比原始PETR（34.8% NDS）提升了3.3个百分点。在nuScenes test set排行榜上，PETRv2取得了50.4% NDS的成绩，展示了时序融合的强大增益。

在BEV分割任务上，PETRv2在nuScenes val set上取得了47.5% mIoU，与同期的BEVFormer等方法具有竞争力。在3D车道线检测任务上，PETRv2在OpenLane数据集上也展示了优异的性能。

消融实验验证了各组件的有效性：时序融合贡献了约3.5% NDS的提升，FGPE贡献了约1.2%的提升。与DETR3D相比，PETRv2在检测精度上领先约7-8个百分点；与BEVFormer相比，PETRv2在不需要显式BEV特征构建的情况下达到了相当的性能。

## 局限性与讨论

1. **计算开销较高**：多帧位置编码和时序融合增加了显存消耗和推理延迟，限制了实时部署的可能性。

2. **时序融合深度有限**：当前设计通常仅融合1-2帧历史信息，更长时序的融合可能带来进一步增益，但也面临计算量和梯度传播的挑战。

3. **BEV分割精度受限**：由于PETRv2并非显式构建BEV表示，其BEV分割质量可能不如专门的BEV方法（如BEVFormer），在需要高精度BEV地图的场景下存在不足。

4. **对远距离目标的感知能力**：受3D采样点分辨率限制，对远距离小目标的检测精度仍有提升空间。

PETRv2展示了Transformer在统一多任务3D感知中的潜力，为后续的多模态融合和大规模预训练工作提供了重要参考。
