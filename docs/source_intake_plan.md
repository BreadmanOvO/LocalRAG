# 首批知识源清单计划（v1）

> 历史规划快照：本文件记录的是 v1.0/v1.1 初期的知识源收集计划，不是当前仓库来源状态的唯一真相。当前实际来源集合与字段口径以 `data/evaluation/shared/source_registry.json` 为准。

## 1. 目标
本清单用于指导 `v1.0 Baseline` 阶段的首批知识源准备工作，服务以下两个直接目标：

1. 为人工 `Gold Set`（30-50 题）提供稳定、可追溯的证据来源。
2. 为首期 `Synthetic Set`（200+ 条）提供覆盖核心主题的来源池。

本阶段不追求全量知识库建设，只追求“够支撑 baseline、证据稳定、后续可扩展”。

## 2. 选择原则
首批知识源按以下原则筛选：

1. **权威性优先**：优先官方文档、标准规范、公开论文与技术报告。
2. **可追溯优先**：优先章节结构清晰、页码或段落可定位的材料。
3. **Gold Set 优先**：优先选择适合直接抽取证据片段的来源。
4. **主题覆盖优先**：至少覆盖感知算法、传感器融合、规划控制、安全规范四类主题。
5. **控制首批规模**：第一轮只准备支撑 baseline 所需的最小高价值集合。
6. **数据集介绍降级**：不同数据集的数据组成、标注形式、任务定义和评测协议差异很大，因此“数据集介绍页”不作为首批主来源大类铺开，最多只保留 1 份代表性数据集技术报告作为补充来源。

## 3. 首批范围总览
建议首批知识源控制在 **11-12 份左右**：

| 类别 | 目标数量 | 优先级 | 主要用途 |
| --- | --- | --- | --- |
| Apollo 官方文档 | 5 篇 | P0 | Gold Set 主力来源，Synthetic Set 主力来源 |
| 标准规范 / 官方规范材料 | 3 份 | P0 | Gold Set 安全题主来源，Synthetic Set 安全长尾补充 |
| 方法论文 / 技术报告 | 3 篇 | P1 | Gold Set 少量综合题，Synthetic Set 中难题主来源 |
| 代表性数据集技术报告 | 1 份 | P1 | 作为补充来源，用于少量数据集类问题，不作为主来源大类扩展 |

说明：
- `P0` 表示必须优先准备。
- `P1` 表示在 P0 形成基本覆盖后立即补齐。
- 数据集介绍页本身不作为 baseline 第一轮的主来源类别，只保留一份代表性技术报告即可。

## 4. 类别拆分

### 4.1 Apollo 官方文档（P0，5 篇）
优先选择结构清晰、能直接支撑问题-证据-答案三元组的核心文档。

建议首批主题：
1. Cyber RT / 运行框架
2. 开发中心 / 总体架构学习路径
3. 开发者平台总体概览
4. 架构版本说明 / 模块栈说明
5. Channel 数据格式 / 模块通信说明

用途建议：
- **Gold Set**：作为最主要来源，优先抽取定义题、原理题、流程题和模块关系题。
- **Synthetic Set**：作为批量扩题的基础来源之一。

筛选要求：
- 页面结构稳定
- 可定位到章节 / 小节
- 描述相对正式，不依赖论坛讨论口径

## 5. 标准规范 / 官方规范材料（P0，3 份）
首批至少覆盖自动驾驶安全规范中的两个核心概念：功能安全与预期功能安全。

建议首批主题：
1. SOTIF（ISO 21448）摘要或权威说明材料
2. ISO 26262 摘要或权威说明材料
3. SAE J3016 或 NHTSA 自动驾驶官方规范性材料（优先选更容易稳定访问的一份）

用途建议：
- **Gold Set**：重点用于定义、比较、场景推理类题目。
- **Synthetic Set**：用于补足 `safety` 主题覆盖。

筛选要求：
- 优先官方或接近官方的表述
- 能清晰区分概念边界
- 可以稳定摘取证据句或证据段

## 6. 方法论文 / 技术报告（P1，3 篇）
这一类不需要首批覆盖太多方向，但要补足技术深度，避免评测集只有概念型问题。

建议首批主题：
1. BEV 感知
2. 多传感器融合
3. Occupancy / motion forecasting / prediction 相关代表性技术材料

用途建议：
- **Gold Set**：少量高质量中难题与综合题。
- **Synthetic Set**：中等与困难样本的主要扩展来源。

筛选要求：
- 优先综述性强、结论表达清晰的材料
- 优先可公开访问、便于后续复核的来源
- 尽量避免首批就纳入过于零散、只能支撑单个术语的问题型论文

## 7. 代表性数据集技术报告（P1，1 份）
这一类只保留 **1 份代表性材料** 作为补充来源，不单独扩成数据集来源大类。

建议用途：
- 仅用于少量“数据组成 / 标注形式 / 任务定义 / 数据集特征”类问题。
- 不应在首批 Gold Set 中占较大比例。

推荐方向：
- Waymo Open Dataset 相关技术报告或官方研究说明

原因：
- 数据集之间的数据组成、标注体系和评测协议差异很大。
- 若过早把数据集介绍当成主来源，会让 Gold Set 偏成 benchmark trivia，而不是领域知识主基准。

## 10. Gold Set 与 Synthetic Set 的来源分工

### 10.1 Gold Set 优先来源顺序
1. Apollo 官方文档
2. 标准规范 / 官方规范材料
3. 方法论文 / 技术报告
4. 代表性数据集技术报告（仅少量）

原因：
- Gold Set 的关键要求是证据稳定、来源明确、问题可核验。
- 官方文档和规范材料更适合作为版本闸门基准。
- 数据集类来源只适合占极少比例。

### 10.2 Synthetic Set 优先来源顺序
1. Apollo 官方文档
2. 方法论文 / 技术报告
3. 标准规范 / 官方规范材料
4. 代表性数据集技术报告（补充）

原因：
- Synthetic Set 更强调覆盖和难度分层。
- 方法论文和技术报告更适合扩展中难题与长尾题。

## 11. 首批不纳入主来源的材料
以下材料暂不作为 baseline 第一批主来源：

- CSDN 技术博客
- 知乎讨论
- 泛行业资讯文章
- 多个数据集的 overview / landing page 大规模平铺收集

原因：
1. 权威性和稳定性较弱，或不适合作为首批主证据源。
2. 对 Gold Set 的证据质量帮助有限。
3. 数据集类来源差异大，首批只保留一份代表性技术报告更稳妥。
4. 更适合作为后续补充材料，而不是 baseline 基准来源。

## 12. 下载与登记规则
首批来源准备完成后，建议立刻做来源登记，避免下载完后再回忆整理。

### 12.1 登记粒度
以“单篇可独立引用的文档”为单位登记。

### 12.2 每份来源至少登记以下信息
- `source_id`
- `title`
- `source_type`
- `language`
- `path_or_url`
- `version`
- `topic_tags`（可选）
- `notes`（可选）

### 12.3 `source_id` 命名建议
- Apollo 文档：`apollo-doc-001`
- 标准规范：`standard-001`
- 方法论文 / 报告：`paper-001`
- 数据集技术报告：`dataset-report-001`

要求：
- 前缀体现来源类别
- 编号稳定，不因顺序调整频繁改名

## 13. 第一轮完成标准
如果满足以下条件，就认为“首批知识源准备完成”：

1. 已收集 **5 篇 Apollo 文档**。
2. 已收集 **3 份标准规范 / 官方规范材料**。
3. 已收集 **3 篇方法论文 / 技术报告**。
4. 已收集 **1 份代表性数据集技术报告**。
5. 四类核心主题（感知、融合、规划控制、安全）均已有来源覆盖。
6. 每份来源都已登记唯一 `source_id`。
7. 至少能从这些来源中抽取出首批 Gold Set 题目的证据片段。

## 14. 后续顺序
首批知识源清单确定后，下一步建议按以下顺序推进：

1. 下载首批核心文档
2. 建立 `source_registry`
3. 落地评估数据契约基础设施
4. 编写 Gold Set 初稿
5. 扩展 Synthetic Set
6. 再接入 `eval_ragas.py` 与 `eval_llm_judge.py`

## 15. 当前建议结论
`v1.0 Baseline` 阶段不需要先做全量知识库采集，但需要先把首批高价值、可追溯、适合评测构建的知识源准备好。

第一轮建议固定为：
- Apollo 官方文档：5 篇
- 标准规范 / 官方规范材料：3 份
- 方法论文 / 技术报告：3 篇
- 代表性数据集技术报告：1 份

合计约 12 份，足够支撑 baseline 第一轮推进。
